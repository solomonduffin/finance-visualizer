## Tech Debt (5 items across 3 phases)

### Phase 4: Frontend Dashboard

1. **UX-03 soft gap:** Dark mode does not visually apply to Settings.tsx and Login.tsx — these pages contain zero `dark:` Tailwind utility classes. The toggle mechanism works (class on `<html>`), but those pages render in light mode only.

2. **UX quality:** Settings page "Back to Dashboard" uses `window.location.href = '/'` instead of React Router `navigate()`, causing a full page reload and re-auth check.

3. **DASH-04 stale type:** `AccountItem` TypeScript interface declares `account_type` but the Go backend serializes as `"type"`. Field is unused in rendering — no runtime impact, but will break if a future feature reads it.

### Phase 2: Data Pipeline

4. **DATA-02 config gap:** `SYNC_HOUR` env var is read by `config.go` but not listed in either `docker-compose.yml` or `docker-compose.prod.yml`. Defaults silently to 6 AM. Operators have no visibility without reading source.

### Phase 1: Foundation

5. **DEPLOY-01 latent risk:** CORS `AllowedOrigins` is hardcoded to `http://localhost:5173` in `router.go`. Works in production because nginx proxies same-origin. Would break if deployed behind a custom domain without nginx.
