    **Step 3: Create AccountsSection component** (`frontend/src/components/AccountsSection.tsx`)
    This is the main UI component. Structure:

    ```
    AccountsSection
      Props: { onAccountRestored?: (names: string[]) => void }
      State: accounts (from getAccounts), editingId, editValue, hiddenExpanded

      AccountGroup (one per type: Liquid, Savings, Investments, Other)
        - Heading with panel type name and account count
        - Droppable zone for drag-and-drop (desktop)
        - List of AccountRow components

      AccountRow (one per visible account)
        - Display: account name (via getAccountDisplayName), balance (via formatCurrency)
        - Below name: subtle gray text showing original SimpleFIN name when display_name is set
        - Pencil icon button -> inline edit mode:
          - Input field with original name as placeholder text
          - Enter saves (calls updateAccount with display_name)
          - Escape cancels (reverts to non-edit state)
          - onBlur saves (same as Enter, per research recommendation)
        - Reset button (only visible when display_name is set): calls updateAccount with display_name=null
        - Hide button (eye icon): calls updateAccount with hidden=true
        - Draggable handle (grip icon, desktop only): for type reassignment
        - Mobile: dropdown select for account_type_override (Liquid, Savings, Investments, Other)
          - Detect mobile via matchMedia('(max-width: 768px)') or Tailwind responsive classes
          - Show dropdown instead of drag handle on mobile

      HiddenAccountsCollapsible
        - Collapsible section at the bottom, default collapsed
        - Header: "Hidden Accounts (N)" with expand/collapse chevron
        - Each hidden account row: grayed out styling, "Unhide" button
        - Unhide calls updateAccount with hidden=false, then refreshes list
    ```

    **Drag-and-drop implementation (desktop):**
    - Use @dnd-kit/react for cross-group drag
    - Each AccountGroup is a droppable container with an id matching the panel type
    - Each AccountRow is a draggable item
    - On drop in a different group: call updateAccount with account_type_override set to the target group type
    - Mapping: Liquid=checking, Savings=savings, Investments=investment, Other=other
    - No within-group reordering (per research recommendation)
    - Show visual drop indicator when dragging over a target group

    **Per-account instant save:**
    - Every action (rename, clear name, hide, unhide, type change) immediately calls the PATCH API
    - Show subtle loading indicator on the row being edited (opacity reduction or spinner)
    - On error, show inline error text and revert local state

    **Data loading:**
    - Call getAccounts() on mount to load all accounts
    - After any updateAccount call, update local state optimistically or re-fetch
    - Prefer optimistic update with error rollback for snappy UX

    **Styling (per user decisions and existing patterns):**
    - Tailwind classes throughout, dark: variants for dark mode
