       ```go
       type netWorthPoint struct {
           Date        string `json:"date"`
           Liquid      string `json:"liquid"`
           Savings     string `json:"savings"`
           Investments string `json:"investments"`
       }

       type netWorthStats struct {
           CurrentNetWorth    string  `json:"current_net_worth"`
           PeriodChangeDollar string  `json:"period_change_dollars"`
           PeriodChangePct    *string `json:"period_change_pct"`    // nil if first total is zero
           AllTimeHigh        string  `json:"all_time_high"`
           AllTimeHighDate    string  `json:"all_time_high_date"`
       }

       type netWorthResponse struct {
           Points []netWorthPoint  `json:"points"`
           Stats  *netWorthStats   `json:"stats"`
       }
       ```
    2. The handler reads `?days=N` query param (default 90, 0 means all). Query:
       ```sql
       SELECT DATE(bs.balance_date) AS bd,
              COALESCE(a.account_type_override, a.account_type) AS effective_type,
              bs.balance
       FROM balance_snapshots bs
       JOIN accounts a ON a.id = bs.account_id
       WHERE a.hidden_at IS NULL
         AND ($days_condition)
       ORDER BY bd, effective_type
       ```
       Where $days_condition is `DATE(bs.balance_date) >= DATE('now', '-N days')` if days > 0, or `1=1` if days == 0.
    3. Accumulate per-day panel totals using the dayAccumulator pattern from history.go. For each date, sum checking+credit into liquid, savings, investment into their respective panels. Use shopspring/decimal for all arithmetic.
    4. Carry forward: Track last known values per panel. When a date has no data for a particular panel, use the last known value for that panel (not zero). This ensures the stacked chart has continuous data.
    5. Compute stats using shopspring/decimal:
       - current_net_worth: sum of all three panels in the last point
       - period_change_dollars: current total minus first point's total
       - period_change_pct: change / first_total * 100; set to nil (JSON null) if first_total is zero
       - all_time_high: max of (liquid+savings+investments) across all points
       - all_time_high_date: date of the all_time_high
    6. Return {"points": [...], "stats": {...}} with Points initialized as `[]netWorthPoint{}` (never nil).
    7. Register in router.go: `r.Get("/api/net-worth", handlers.GetNetWorth(database))` in the protected route group.
    8. Write comprehensive tests in networth_test.go using existing setupFinanceTestDB helper.

    NOTE: This endpoint does NOT account for group panel_type reassignment yet (that is handled when Plan 01's group modifications to the accounts layer are available). For now, use each account's own effective_type. Group-aware panel assignment will be addressed in a follow-up if needed, but since this endpoint queries balance_snapshots directly with effective_type, and the group panel_type affects the display layer (which panel to show the group in) rather than the underlying balance data, this approach is correct.
