**Data fetching on mount:**
```typescript
useEffect(() => {
  async function load() {
    try {
      setLoading(true)
      const [settingsRes, netWorthRes] = await Promise.all([
        getProjectionSettings(),
        getNetWorth('6m'),  // ~6 months of historical data
      ])
      setAccounts(settingsRes.accounts)
      setIncome(settingsRes.income)
      // Transform net worth points to total net worth per date
      setHistoricalData(
        netWorthRes.points.map(p => ({
          date: p.date,
          value: p.liquid + p.savings + p.investments,
        }))
      )
    } catch {
      setError('Something went wrong loading projection data')
    } finally {
      setLoading(false)
    }
  }
  load()
}, [])
```

**Projection calculation (via useMemo):**
Convert the ProjectionAccountSetting[] and income into the types expected by calculateProjection:
- For each account: map to AccountProjection with `hasHoldings: account.holdings.length > 0`, parse balance/apy as numbers
- For accounts with holdings: map each holding to HoldingProjection, parse market_value/apy/allocation as numbers
- Parse allocation_json to get allocation percentages for accounts without holdings
- Call calculateProjection with the mapped data, income settings (parsed to numbers), and horizonYears
- Wrap in useMemo that depends on accounts, income, horizonYears

**Debounced auto-save:**
```typescript
const saveTimeoutRef = useRef<number>()

function debouncedSaveSettings(updatedAccounts: ProjectionAccountSetting[]) {
  clearTimeout(saveTimeoutRef.current)
  saveTimeoutRef.current = window.setTimeout(async () => {
    try {
      await saveProjectionSettings({
        accounts: updatedAccounts.map(a => ({
          account_id: a.account_id,
          apy: a.apy,
          compound: a.compound,
          included: a.included,
        })),
        holdings: updatedAccounts.flatMap(a =>
          a.holdings.map(h => ({
            holding_id: h.holding_id,
            account_id: a.account_id,
            apy: h.apy,
            compound: h.compound,
            included: h.included,
            allocation: h.allocation,
          }))
        ),
      })
    } catch {
      // Show toast: "Failed to save settings"
    }
  }, 500)
}

function debouncedSaveIncome(updatedIncome: ProjectionIncomeSettings) {
  clearTimeout(incomeTimeoutRef.current)
  incomeTimeoutRef.current = window.setTimeout(async () => {
    try {
      await saveIncomeSettings(updatedIncome)
    } catch {
      // Show toast: "Failed to save settings"
    }
  }, 500)
}
```
Cleanup: `useEffect(() => () => { clearTimeout(saveTimeoutRef.current); clearTimeout(incomeTimeoutRef.current) }, [])`

**onChange handlers:**
Each handler:
1. Updates local state (setAccounts or setIncome)
2. Calls the debounced save function
3. The useMemo recalculation triggers automatically from state change

Example for onApyChange:
```typescript
const handleApyChange = (accountId: string, apy: string) => {
  setAccounts(prev => {
    const next = prev.map(a =>
      a.account_id === accountId ? { ...a, apy } : a
    )
    debouncedSaveSettings(next)
    return next
  })
}
```

Similar handlers for: onHoldingApyChange, onCompoundChange, onHoldingCompoundChange, onIncludeChange (with master cascade for investment accounts), onHoldingIncludeChange, income onToggle, onAnnualIncomeChange, onMonthlySavingsPctChange, onAllocationChange.

**Allocation targets derivation:**
Compute allocation targets from accounts state:
- For included accounts without holdings: add `{ id: "acct:${account_id}", name: account_name, percentage: fromAllocationJson }`
- For included holdings: add `{ id: "hold:${holding_id}", name: description, percentage: holding.allocation }`
- When allocation changes, update allocation_json in income state
