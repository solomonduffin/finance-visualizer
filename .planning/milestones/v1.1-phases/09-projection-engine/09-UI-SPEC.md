# Phase 9 -- UI Design Contract

> Visual and interaction contract for the projection engine: forward-looking net worth chart with solid-to-dashed transition, per-account/per-holding rate configuration table, compound/simple toggles, income allocation modeling, and time horizon selector. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-rolled Tailwind components) |
| Icon library | inline SVG (16x16, stroke-based, matching GroupRow chevron and NavBar icon patterns) |
| Font | System font stack (Tailwind default: ui-sans-serif, system-ui, sans-serif) |

**Note:** Continues the hand-rolled Tailwind convention established in v1.0 and maintained through Phases 6-8. Phase 9 adds 1 new page (Projections) and several new components (ProjectionChart, RateConfigTable, IncomeModelingSection, HorizonSelector). No shadcn or third-party component library.

**Focal points:**
- Projection chart: The combined historical + projected net worth line chart is the hero visual. Solid-to-dashed transition at "now" is the defining visual feature.
- Rate configuration table: The primary interaction surface. Users spend most time adjusting APY inputs and toggles here, with live chart feedback.
- Income modeling: Secondary collapsible section for power users. Defaults to collapsed.

## ProjectionChart Specification

**Purpose:** Combined historical + projected net worth line chart. This is the hero visual of the page.

**Chart library:** Recharts (consistent with BalanceLineChart and StackedAreaChart)

**Layout:**
```
[Chart card: bg-white dark:bg-gray-800 rounded-xl shadow-sm border border-gray-100 dark:border-gray-700 p-6]
  [ResponsiveContainer width="100%" height={400}]
    [ComposedChart or LineChart]
      [Historical line: solid, gray-700/gray-300]
      [Projection line: dashed, blue-500/blue-400, with area fill]
      [Vertical "Now" reference line: dashed, gray-400/gray-500]
      [XAxis: date labels]
      [YAxis: currency abbreviated ($50k, $100k)]
      [Tooltip]
      [CartesianGrid: strokeDasharray="3 3", gray-200/gray-700 at 30% opacity]
```

**Data structure:**
- Each data point: `{ date: string, historical: number | null, projected: number | null }`
- Historical points have `historical` set, `projected` null
- The "now" point has BOTH `historical` and `projected` set (equal to current net worth) -- this creates the visual connection
- Projection points have `projected` set, `historical` null

**Line styling:**
- Historical: `<Line dataKey="historical" stroke={isDark ? '#d1d5db' : '#374151'} strokeWidth={2} dot={false} connectNulls={false} />`
- Projection: `<Line dataKey="projected" stroke={isDark ? '#60a5fa' : '#3b82f6'} strokeWidth={2} strokeDasharray="8 4" dot={false} connectNulls={false} />`
- Projection area fill: `<Area dataKey="projected" fill={isDark ? 'rgba(96,165,250,0.15)' : 'rgba(59,130,246,0.2)'} stroke="none" />`

**"Now" reference line:**
- Recharts `<ReferenceLine x={nowDate} stroke={isDark ? '#6b7280' : '#9ca3af'} strokeDasharray="4 4" />`
- Label: `<Label value="Now" position="top" fill={isDark ? '#9ca3af' : '#6b7280'} fontSize={12} fontWeight={600} />`

**X-axis formatting:**
- Historical dates: "Mar 15" style (month + day) -- matching existing charts
- Projection dates: "Jun '26", "2027", "2030" -- year-level labels for long horizons
- Tick interval: auto-calculated based on horizon length. Short horizons (1y) show monthly ticks. Long horizons (20y) show yearly ticks.

**Y-axis formatting:**
- Reuse abbreviated format from StackedAreaChart: `$50k`, `$100k`, `$1M`
- Domain: auto from data with 10% padding

**Tooltip:**
- Container: `bg-white dark:bg-gray-800 border border-gray-200 dark:border-gray-700 rounded-lg shadow-lg p-3`
- Date label: `text-xs font-semibold text-gray-500 dark:text-gray-400 mb-1`
- Value: `text-sm font-semibold text-gray-900 dark:text-gray-100` with full currency format ($123,456.78)
- Shows "Historical" or "Projected" label in `text-xs` above the value to distinguish which line segment the user is hovering

**Chart height:** 400px (matching NetWorth page chart height)

**Empty chart state:** If no historical data or projection data exists, the chart card displays the message "No data to project. Sync your accounts to see projections." centered inside the 400px container.

## HorizonSelector Specification

**Purpose:** Time horizon selector for the projection period. Adapts the TimeRangeSelector segmented control pattern with projection-specific presets and a Custom year input.

**Layout:**
```
[Segmented control: inline-flex rounded-lg bg-gray-100 dark:bg-gray-800 p-1]
  [1y] [5y] [10y] [20y] [Custom]
[Custom year input: conditionally revealed below]
```

**Presets:**
```typescript
const HORIZONS = [
  { label: '1y', years: 1 },
  { label: '5y', years: 5 },
  { label: '10y', years: 10 },
  { label: '20y', years: 20 },
  { label: 'Custom', years: -1 }, // sentinel for custom mode
] as const
```

**Button styling:** Identical to TimeRangeSelector pattern:
- Active: `bg-blue-600 text-white font-semibold shadow-sm`
- Inactive: `bg-transparent text-gray-600 dark:text-gray-400 hover:text-gray-900 dark:hover:text-gray-200 font-normal`
- All: `px-3 py-2 text-sm rounded-md transition-colors`

**Accessibility:** `role="radiogroup"`, `aria-label="Select projection horizon"`, each button `role="radio"`, `aria-checked`

**Custom year input:**
- Revealed when "Custom" is the active selection
- Appears inline to the right of the segmented control (desktop) or below it (mobile)
- Input: `<input type="number" min="1" max="50" step="1">`
- Style: `w-20 px-3 py-2 border border-gray-300 dark:border-gray-600 rounded-lg text-sm text-gray-900 dark:text-gray-100 bg-white dark:bg-gray-800 focus:outline-none focus:ring-2 focus:ring-blue-500`
- Suffix label: "years" in `text-sm text-gray-500 dark:text-gray-400 ml-2`
- Default custom value: 5 years
- Layout: `flex items-center gap-2 mt-2 md:mt-0 md:ml-3`

## RateConfigTable Specification

**Purpose:** Account rate configuration table grouped by panel type. Users set APY, compound/simple toggle, and include/exclude for each account (and per-holding for investment accounts with holdings data).

**Card container:** `bg-white dark:bg-gray-800 rounded-xl shadow-sm border border-gray-100 dark:border-gray-700 p-6`

**Card heading:** "Projection Settings" -- `text-lg font-semibold text-gray-900 dark:text-gray-100 mb-4`

**Table layout:** Uses CSS grid for alignment, not `<table>` element (better responsive behavior).

**Column headers:**
```
[Account]           [APY %]  [Compound]  [Include]
```
- Style: `text-xs font-semibold uppercase tracking-wide text-gray-500 dark:text-gray-400`
- Grid: `grid grid-cols-[1fr_80px_80px_64px] gap-2 items-center pb-3 border-b border-gray-200 dark:border-gray-700`
- Column widths: account name (flex), APY input (80px), compound toggle (80px), include checkbox (64px)

**Panel group headers:**
- One header per panel type that has accounts: Liquid, Savings, Investments
- Style: `flex items-center gap-2 pt-4 pb-2`
- Accent dot: `w-2 h-2 rounded-full` with panel color from `PANEL_COLORS[panelKey].accent`
- Label: `text-sm font-semibold uppercase tracking-wide` with panel color applied via `style={{ color: panelColor }}`

**Account row (standard -- no holdings):**
```
grid grid-cols-[1fr_80px_80px_64px] gap-2 items-center py-2 border-b border-gray-100 dark:border-gray-700/50
```
- Account name: `text-sm text-gray-800 dark:text-gray-200 truncate` -- uses `getAccountDisplayName()`
- APY input: `<input type="text" inputMode="decimal">`
  - Style: `w-full px-2 py-1.5 border border-gray-300 dark:border-gray-600 rounded-md text-sm text-right text-gray-900 dark:text-gray-100 bg-white dark:bg-gray-800 placeholder-gray-400 dark:placeholder-gray-500 focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-transparent`
  - Placeholder: "0.0"
  - Suffix "%" rendered as static text to the right of the input: `text-xs text-gray-500 dark:text-gray-400`
  - Fires onChange with 500ms debounce for live chart update
- Compound toggle: miniature toggle switch (smaller than DashboardPreferences toggle)
  - Size: `w-8 h-[18px]` with `w-[14px] h-[14px]` knob
  - On state (compound): `bg-blue-600 dark:bg-blue-500`
  - Off state (simple): `bg-gray-300 dark:bg-gray-600`
  - `role="switch"`, `aria-checked`, `aria-label="Compound interest for {accountName}"`
  - Tooltip on hover: "Compound" when on, "Simple" when off -- using `title` attribute
- Include checkbox: standard checkbox appearance
  - Style: `w-4 h-4 rounded border border-gray-300 dark:border-gray-600 text-blue-600 focus:ring-2 focus:ring-blue-500 accent-blue-600`
  - `aria-label="Include {accountName} in projection"`
  - Checked by default for all accounts

**Investment account row WITH holdings (expandable):**
- Same grid as standard account row
- Account name area includes chevron icon (16x16, matching GroupRow pattern)
  - `flex items-center gap-1 cursor-pointer`
  - Chevron rotates 90 degrees on expand
  - Click toggles expansion
  - `role="button"`, `tabIndex={0}`, `aria-expanded`, `aria-controls`
  - Enter/Space keyboard toggle
- APY input: hidden (or disabled with dash) -- rates are per-holding, not per-account
- Compound toggle: hidden -- per-holding
- Include checkbox: acts as master toggle. Unchecking excludes all holdings. Checking includes all holdings.
- Current balance shown after account name: `text-xs text-gray-500 dark:text-gray-400 ml-2` -- "{formatCurrency(balance)}"

**Holdings sub-row (within expanded investment account):**
```
grid grid-cols-[1fr_80px_80px_64px] gap-2 items-center py-2 pl-8
```
- Holding name: `text-sm text-gray-600 dark:text-gray-300 truncate`
- Holding value: `text-xs text-gray-500 dark:text-gray-400` shown below holding name
- APY input: same style as account-level APY input
- Compound toggle: same miniature toggle as account-level
- Include checkbox: individual holding include/exclude
- Expansion animation: `max-height` transition 200ms ease, `overflow: hidden`, `motion-reduce:transition-none` -- matching GroupRow pattern

**Investment account WITHOUT holdings:**
- Rendered as standard account row (no chevron, no expand)
- APY input at account level
- Compound toggle at account level

**Rate table states:**
1. **Loading:** Skeleton rows -- 6 rows of `h-10 bg-gray-200 dark:bg-gray-700 rounded animate-pulse`
2. **Empty:** "No accounts found. Sync your accounts first." in muted text
3. **Populated:** Full table with grouped accounts

## IncomeModelingSection Specification

**Purpose:** Collapsible section for optional income modeling. When enabled, monthly savings are allocated to included accounts/holdings and added to the projection.

**Card container:** `bg-white dark:bg-gray-800 rounded-xl shadow-sm border border-gray-100 dark:border-gray-700`

**Collapsed header (default state):**
```
[flex items-center justify-between p-6 cursor-pointer]
  [flex items-center gap-2]
    [ChevronIcon: 16x16, right-pointing]
    [h2: "Income Modeling" -- text-lg font-semibold text-gray-900 dark:text-gray-100]
  [Toggle switch: enable/disable income modeling]
```
- Clicking the chevron/heading area expands the section
- Toggle switch independently enables/disables income modeling in the projection calculation
- Toggle: reuses DashboardPreferences toggle pattern (`role="switch"`, `aria-checked`, `w-10 h-[22px]`)
- `aria-label="Enable income modeling"`
- When toggle is off AND section is collapsed: section appears at reduced opacity (`opacity-60`)

**Expanded state:**
```
[Header row (same as collapsed)]
[Content area: px-6 pb-6 pt-0]
  [Income inputs: flex flex-col gap-4 md:flex-row md:gap-6]
    [Annual Income field]
    [Monthly Savings % field]
    [Computed monthly allocation display]
  [Divider: border-t border-gray-200 dark:border-gray-700 my-4]
  [Allocation heading + helper text]
  [Allocation rows]
  [Allocation sum validation]
```

**Expansion animation:** `max-height` transition 300ms ease, `overflow: hidden`, `motion-reduce:transition-none`

**Income input fields:**

| Field | Label | Type | Placeholder | Width |
|-------|-------|------|-------------|-------|
| Annual Income | Annual Income | text, inputMode="decimal" | 0 | `flex-1` or `w-48` on desktop |
| Monthly Savings % | Monthly Savings | text, inputMode="decimal" | 0 | `w-24` |

- Field style: same as rate table APY input but full-width within their containers
  - `px-3 py-2 border border-gray-300 dark:border-gray-600 rounded-lg text-sm text-gray-900 dark:text-gray-100 bg-white dark:bg-gray-800 placeholder-gray-400 dark:placeholder-gray-500 focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-transparent`
- Annual Income input: prepend "$" as static label inside input container (using `flex items-center` wrapper with dollar sign in `text-sm text-gray-500 dark:text-gray-400`)
- Monthly Savings input: append "%" as static suffix
- Computed monthly allocation: `text-sm text-gray-600 dark:text-gray-300` -- displays "{formatCurrency(annualIncome / 12 * savingsPercent / 100)} / month to allocate" -- updates live

**Allocation heading:**
- Text: "Savings Allocation" -- `text-sm font-semibold text-gray-700 dark:text-gray-300 mb-1`
- Helper: "Distribute monthly savings across your included accounts. Must total 100%." -- `text-xs text-gray-500 dark:text-gray-400 mb-3`

**Allocation rows (AllocationRow component):**
- One row per included account (or per included holding for investment accounts with holdings)
- Excluded accounts do not appear in allocation
- Layout: `flex items-center gap-3 py-2`
  - Account/holding name: `text-sm text-gray-700 dark:text-gray-300 truncate flex-1`
  - Percentage input: `<input type="text" inputMode="decimal">` -- `w-16 px-2 py-1.5 border border-gray-300 dark:border-gray-600 rounded-md text-sm text-right text-gray-900 dark:text-gray-100 bg-white dark:bg-gray-800 focus:outline-none focus:ring-2 focus:ring-blue-500`
  - "%" suffix: `text-xs text-gray-500 dark:text-gray-400`
  - Visual bar: horizontal bar showing relative allocation
    - Track: `h-1.5 flex-1 bg-gray-200 dark:bg-gray-700 rounded-full`
    - Fill: `h-1.5 bg-blue-500 dark:bg-blue-400 rounded-full transition-all duration-200` with width set to `{percentage}%`

**Allocation sum validation:**
- Displayed below all allocation rows: `flex items-center justify-end gap-2 mt-3`
- Sum display: `text-sm font-semibold`
  - When sum equals 100: `text-green-600 dark:text-green-400` -- "Total: 100%"
  - When sum does not equal 100: `text-red-600 dark:text-red-400` -- "Total: {sum}% (must equal 100%)"
- Projection recalculation is suppressed while allocation sum is not 100% -- chart shows last valid state

**When income toggle is off:**
- The expanded section content area is visually dimmed (`opacity-50 pointer-events-none`)
- All inputs disabled
- Chart ignores income contribution

## Projections Page Specification

**Route:** `/projections`

**Page layout:**
```
[NavBar]
[Page container: min-h-screen bg-gray-50 dark:bg-gray-900]
  [max-w-5xl mx-auto px-4 py-6]
    [h1: "Projections"]
    [HorizonSelector (mb-4)]
    [ProjectionChart card]
    [RateConfigTable card (mt-6)]
    [IncomeModelingSection card (mt-6)]
```

**Page background:** `min-h-screen bg-gray-50 dark:bg-gray-900` (matches Dashboard, Net Worth, Alerts)

**Container:** `max-w-5xl mx-auto px-4 py-6` -- same width as Dashboard. The rate table benefits from horizontal space for the grid columns.

**Header:** `text-2xl font-semibold text-gray-900 dark:text-gray-100 mb-6`

**States:**
1. **Loading:** Skeleton chart (400px gray rectangle) + skeleton table rows (6 rows) -- both with `animate-pulse`
2. **Empty (no accounts):** EmptyState component -- heading "No accounts to project", body "Sync your financial accounts to start building projections.", CTA "Go to Settings" linking to /settings
3. **Error:** Centered error message with retry button (matches Net Worth error pattern)
4. **Populated:** Full page with chart, rate table, income section

**Data flow:**
- On mount: fetch accounts (with holdings for investment accounts) and any persisted projection settings
- Projection calculation runs client-side (no server-side projection endpoint needed for chart rendering)
- Rate/toggle/checkbox changes fire with 500ms debounce, trigger recalculation, chart re-renders
- Settings persist via API call after each debounced change (auto-save, no explicit "Save" button for rates)
- Income settings also auto-save on change with debounce

## Interaction Contracts

### Live Chart Update (Rate/Toggle/Checkbox Changes)
- **Trigger:** User changes any APY input, compound toggle, include checkbox, or income parameter
- **Response:** 500ms debounce timer starts. On debounce expiry, projection recalculates client-side and chart re-renders smoothly.
- **Visual feedback:** No explicit loading indicator for chart recalculation (it should be near-instant since it runs client-side). If recalculation takes > 100ms, show a subtle opacity pulse on the chart.
- **Auto-save:** After debounce, changed settings are persisted to the server via API PATCH call in the background. No save button, no toast on success. Toast on save failure: "Failed to save settings" using Toast component.

### Horizon Selector
- **Trigger:** Click a preset button (1y, 5y, 10y, 20y) or select Custom and type a year value
- **Response:** Chart re-renders with new projection endpoint. X-axis scale adjusts. Preset buttons update active state immediately.
- **Custom input:** Appears when "Custom" is active. Typing a number triggers chart update on blur or after 500ms debounce. Min 1, max 50 years.

### Rate Table Account Expand/Collapse (Investment Holdings)
- **Trigger:** Click the chevron or account name on an investment account row with holdings
- **Response:** Chevron rotates 90 degrees, holdings sub-rows slide in below the account row
- **Animation:** `max-height` transition 200ms ease, `overflow: hidden`, `motion-reduce:transition-none` -- matching GroupRow pattern
- **State:** Multiple accounts can be expanded simultaneously
- **Keyboard:** Enter/Space on the account row area toggles expansion

### Income Modeling Section Expand/Collapse
- **Trigger:** Click the chevron or "Income Modeling" heading
- **Response:** Chevron rotates, content area slides in/out
- **Animation:** `max-height` transition 300ms ease, `overflow: hidden`, `motion-reduce:transition-none`
- **Independent from toggle:** Section can be expanded while toggle is off (content dimmed). Toggle can be on while section is collapsed (income still contributes to projection).

### Income Modeling Toggle
- **Trigger:** Click the toggle switch in the income section header
- **Response:** Immediate optimistic UI update. Chart recalculates (with or without income contribution). Toggle state auto-saves.
- **Accessibility:** `role="switch"`, `aria-checked`, `aria-label="Enable income modeling"`

### Include/Exclude Checkbox
- **Trigger:** Click checkbox on an account row
- **Response:** Immediate visual update. Chart recalculates excluding/including that account. For investment accounts with holdings, checking/unchecking the master checkbox cascades to all holdings.
- **Allocation impact:** When an account is excluded, its allocation row disappears from the income section. Remaining allocations may no longer sum to 100% -- validation message updates.

### Allocation Percentage Input
- **Trigger:** User types a percentage value in an allocation row
- **Response:** Visual bar updates width immediately. Sum validation updates. Chart recalculates after 500ms debounce (only if sum is 100%).
- **Constraint:** Sum must equal 100% for income to be applied to the projection. When sum is not 100%, the chart shows the last valid projection state and the validation error is displayed.
