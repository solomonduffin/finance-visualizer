Create `internal/alerts/` package with 4 source files and 4 test files:

**crypto.go** (`package alerts`):
- `func DeriveKey(secret string) []byte` - SHA-256 hash of secret, returns 32-byte slice for AES-256
- `func Encrypt(plaintext string, key []byte) (string, error)` - AES-256-GCM encrypt, prepend nonce, base64 encode
- `func Decrypt(encoded string, key []byte) (string, error)` - base64 decode, extract nonce, AES-256-GCM decrypt
- Use exact implementation from 08-RESEARCH.md Pattern 4

**evaluator.go** (`package alerts`):
- `type Operand struct` with fields: Type string `json:"type"`, Ref string `json:"ref"`, Label string `json:"label"`, Operator string `json:"operator"`
- `type Environment struct` with fields: Liquid float64 `expr:"liquid"`, Savings float64 `expr:"savings"`, Investments float64 `expr:"investments"`, NetWorth float64 `expr:"net_worth"`, Accounts map[string]float64 `expr:"accounts"`, Groups map[string]float64 `expr:"groups"`
- `func CompileOperands(operandsJSON []byte, comparison string, threshold string) (string, error)`:
  1. Unmarshal operandsJSON into []Operand
  2. Validate: at least 1 operand, comparison in {<, <=, >, >=, ==}
  3. Build arithmetic expression string: for bucket refs use field name directly (e.g. "liquid"), for account refs use `accounts["id"]`, for group refs use `groups["id"]`
  4. Wrap multi-term arithmetic in parentheses: `(term1 + term2 - term3)`
  5. Append comparison and threshold: `(terms) < 5000`
  6. Call expr.Compile with expr.Env(Environment{}) and expr.AsBool() to validate
  7. Return the expression string
- `func Validate(expression string) error` - compile-only check against Environment{}
- `func Evaluate(expression string, env Environment) (bool, error)` - compile and run, return bool result
- `func BuildEnvironment(ctx context.Context, db *sql.DB) (*Environment, error)`:
  1. Query latest balance per visible account with effective panel type (COALESCE account_type_override, then check group panel_type): same aggregation logic as summary endpoint
  2. Sum into bucket totals (liquid = checking - credit, savings, investments)
  3. Populate Accounts map (account ID -> float64 balance)
  4. Populate Groups map (group ID string -> summed member balance float64)
  5. NetWorth = Liquid + Savings + Investments
  6. Include hidden accounts in the Accounts map (so existing rules referencing soft-deleted accounts still evaluate; use balance 0 if no snapshot)

**engine.go** (`package alerts`):
- `func NextState(current string, conditionMet bool, notifyRecovery bool) (newState string, shouldNotify bool)`:
  State transition table exactly as in 08-RESEARCH.md Pattern 2
- `func EvaluateAll(ctx context.Context, db *sql.DB, jwtSecret string) error`:
  1. Build environment via BuildEnvironment
  2. Query all enabled alert_rules
  3. For each rule: Evaluate expression against environment
  4. Call NextState with current last_state, evaluation result, notify_on_recovery flag
  5. If state changed: UPDATE alert_rules SET last_state=?, last_eval_at=CURRENT_TIMESTAMP, last_value=?, updated_at=CURRENT_TIMESTAMP
  6. If shouldNotify: INSERT alert_history row, then load SMTP config from settings table, decrypt password using DeriveKey(jwtSecret), call SendAlert (best-effort, log errors)
  7. If no state change: just UPDATE last_eval_at and last_value
  8. Wrap email sending with 10-second context timeout
  9. Never return error from individual rule failures (log and continue)
  10. Return nil unless BuildEnvironment fails

**notifier.go** (`package alerts`):
- `type SMTPConfig struct` with Host, Port int, Username, Password, FromAddress, ToAddress string
- `type AlertDetail struct` with RuleName, Status ("triggered"/"recovered"), ComputedValue, Threshold, Comparison string, Operands []Operand, OperandValues map[string]string, Timestamp time.Time
- `func FormatSubject(ruleName, state string) string` - returns `[Finance Alert] {name}` or `[Finance Alert] {name} -- recovered`
- `func FormatAlertBody(detail AlertDetail) string`:
  Plain text body with sections:
  ```
  Alert: {rule name}
  Status: TRIGGERED (or RECOVERED)

  Computed Value: ${value}
  Threshold: {comparison} ${threshold}
  Time: {timestamp in RFC822 format}

  Account Breakdown:
  - {operand label}: ${value}
  - {operand label}: ${value}

  --
  Finance Visualizer
  ```
- `func SendAlert(ctx context.Context, cfg SMTPConfig, detail AlertDetail) error`:
  Use go-mail: NewMsg, From, To, Subject(FormatSubject), SetBodyString(TypeTextPlain, FormatAlertBody). Create client with Host, Port, SMTPAuthAutoDiscover, Username, Password. DialAndSendWithContext with 10s timeout.
- `func LoadSMTPConfig(ctx context.Context, db *sql.DB, jwtSecret string) (*SMTPConfig, error)`:
  Query settings table for keys: smtp_host, smtp_port, smtp_username, smtp_password, smtp_from, smtp_to. Decrypt smtp_password using DeriveKey(jwtSecret). Return nil if smtp_host not configured. Parse smtp_port as int (default 587).

Write tests first (RED), then implement (GREEN). Tests for notifier_test.go should test FormatAlertBody and FormatSubject (pure functions). Do NOT test actual SMTP sending -- that requires a real server. For engine_test.go, use a test SQLite database (same pattern as handlers_test.go with db.Open + db.Migrate).
