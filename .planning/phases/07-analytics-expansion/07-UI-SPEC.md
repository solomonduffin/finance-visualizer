# Phase 7 — UI Design Contract

> Visual and interaction contract for account groups and net worth drill-down page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-rolled Tailwind components) |
| Icon library | inline SVG (16x16, stroke-based, matching SyncHistory/AccountsSection pattern) |
| Font | System font stack (Tailwind default: ui-sans-serif, system-ui, sans-serif) |

**Note:** Continues the hand-rolled Tailwind convention established in v1.0 and maintained through Phase 6. Phase 7 adds 5 new components and modifies 5 existing ones. No shadcn or third-party component library.

**Focal points:**
- Dashboard: PanelCard total balance (24px font-semibold) remains the primary anchor; group rows are visually subordinate to it.
- Net Worth page: Current net worth stat (32px font-semibold) is the primary anchor; the stacked area chart is the secondary focal point.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chevron margin, inline padding, `gap-1` |
| sm | 8px | Compact element spacing, account indent, `gap-2`, `py-2` |
| md | 16px | Default section padding, stat card padding, `p-4`, `mb-4` |
| lg | 24px | Card padding, chart margins, `p-6`, `mb-6` |
| xl | 32px | Stats bar gap between stat items, `gap-8` |
| 2xl | 48px | Page vertical padding, `py-12` |
| 3xl | 64px | Not used in this phase |

Exceptions: none

**Phase-specific spacing rules:**
- Group member indent: `pl-6` (24px) left padding for member accounts nested under a group row
- Segmented control button padding: `px-3 py-1.5` (12px horizontal, 6px vertical) for touch-friendly targets
- Stats bar: `gap-8` (32px) between stat items on desktop; stacks vertically on mobile with `gap-4` (16px)
- Stacked area chart container: `h-[400px]` on desktop, `h-[280px]` on mobile
- Nav link gap: `gap-3` (12px) between nav items, consistent with existing NavBar

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (normal) | 1.5 | `text-sm` |
| Heading | 18px | 600 (semibold) | 1.4 | `text-lg font-semibold` |

**Permitted weights:** 400 (`font-normal`) and 600 (`font-semibold`) only. No other weights may be used in this phase.

**Derived roles (using the 2 declared weights):**
- Label: 12px at weight 600 -- `text-xs font-semibold` (panel card type labels, chart axis labels)
- Display: 24px at weight 600 -- `text-2xl font-semibold` (panel card total balance)
- Page Title: 24px at weight 600 -- `text-2xl font-semibold` (page heading, e.g., "Net Worth")
- Stat Value: 32px at weight 600 -- `text-3xl font-semibold` (current net worth stat, primary page anchor)
- Stat Label: 12px at weight 600 -- `text-xs font-semibold uppercase tracking-wide` (stat description labels)

**Phase-specific typography:**
- Group row name: `text-sm font-semibold` (14px, weight 600) -- distinguishes group from individual accounts
- Group member name: `text-sm font-normal` (14px, weight 400) -- same as existing individual account names
- Group summed balance: `text-sm font-medium` -- wait, use `font-semibold` for consistency: `text-sm font-semibold`
- Net worth stat value (current): `text-3xl font-semibold` (32px) -- primary focal point of the page
- Net worth stat value (period change, ATH): `text-lg font-semibold` (18px) -- secondary stats
- Net worth stat label: `text-xs font-semibold uppercase tracking-wide text-gray-500 dark:text-gray-400`
- Segmented control label: `text-sm font-semibold` (14px, weight 600) for active; `text-sm font-normal` for inactive
- Chart tooltip date: `text-sm font-semibold` (14px, weight 600)
- Chart tooltip values: `text-sm font-normal` (14px, weight 400)
- Chart axis ticks: 11px (Recharts tick prop, matching BalanceLineChart)
- Nav link "Net Worth": `text-sm font-medium` -- matches existing "Settings" nav link style

### GroupRow Specification

**Layout:** Renders as a row within PanelCard's account list area, visually distinguished from individual accounts.

```
Collapsed:  [>] Coinbase          $5,230.00  +3.2%
Expanded:   [v] Coinbase          $5,230.00  +3.2%
                  CB Wallet 1     $3,000.00
                  CB Wallet 2     $2,230.00
```

**Row structure (collapsed):**
```
<div class="flex justify-between items-center text-sm py-1 cursor-pointer">
  <div class="flex items-center gap-1">
    <ChevronIcon expanded={false} />
    <span class="font-semibold text-gray-800 dark:text-gray-100 truncate">Coinbase</span>
  </div>
  <div class="flex items-center gap-2 shrink-0">
    <span class="font-semibold text-gray-800 dark:text-gray-200">$5,230.00</span>
    <GrowthBadge ... />
  </div>
</div>
```

**Member rows (expanded):**
```
<div class="pl-6">
  <div class="flex justify-between items-center text-sm py-0.5">
    <span class="text-gray-600 dark:text-gray-300 truncate pr-2">CB Wallet 1</span>
    <span class="text-gray-800 dark:text-gray-200 font-medium shrink-0">$3,000.00</span>
  </div>
</div>
```

**States:**
1. **Collapsed (default):** Chevron pointing right, members hidden
2. **Expanded:** Chevron rotated 90 degrees, members visible below with `pl-6` indent
3. **No members:** Should not occur (empty groups are auto-deleted)

**Animation:** Chevron rotation: `transition-transform duration-200 motion-reduce:transition-none` (matches SyncHistory ChevronIcon). Member list: CSS `max-height` transition from 0 to content height, `overflow: hidden`, 200ms ease, `motion-reduce:transition-none`.

**Interaction:**
- Click anywhere on the group row toggles expand/collapse
- Multiple groups can be expanded simultaneously (independent state per group, unlike SyncHistory which allows only one)
- `role="button"`, `tabIndex={0}`, `aria-expanded`, `aria-controls` on the group row
- Keyboard: Enter/Space toggles expansion

**ChevronIcon:** Reuse the exact SVG from SyncHistory.tsx (16x16, stroke-based, right-pointing chevron with `rotate-90` when expanded).

**Axes:**
- XAxis: `dataKey="date"`, `tick={{ fontSize: 11 }}`, stroke `gray-500`/`gray-400` (dark)

```
Container: bg-white dark:bg-gray-800 border border-gray-200 dark:border-gray-700 rounded-lg shadow-md p-3 text-sm
```

**Legend:** Not rendered separately -- the tooltip provides per-panel context on hover. The stat labels and the stacked visual already communicate which color maps to which panel. If needed in the future, a legend can be added below the chart matching the NetWorthDonut legend pattern.

### Account Groups in Settings Specification

**"+ New Group" button:**
- Placement: Top of the Accounts section, right-aligned next to the section heading
- Style: `text-sm font-semibold text-blue-600 dark:text-blue-400 hover:text-blue-700 dark:hover:text-blue-300 transition-colors cursor-pointer`
- Text: "+ New Group"
- No background/border -- text-only link style (lightweight, not a heavy CTA)

**Group creation flow:**
1. Click "+ New Group"
2. Inline text input appears at the top of the accounts list with autofocus
3. User types group name and presses Enter to confirm (or Escape to cancel)
4. Group is created via API with default panel_type matching the section it was created in
5. Input replaced by the new group entry

**Inline name input:**
- Style: `text-sm border border-blue-400 rounded px-2 py-1 focus:outline-none focus:ring-2 focus:ring-blue-500`
- Placeholder: "Group name"

**Group display in Settings:**
```
[Liquid section]
  [Group: "Main Bank" -- bold name, drag handle, chevron]
    [Account: Checking -- indented, draggable]
    [Account: Credit Card -- indented, draggable]
  [Account: Standalone -- standard, draggable]

[Savings section]
  [Account: Savings 1 -- standard, draggable]
```

**Group row in Settings:**
- Drag handle (same grip icon from existing accounts)
- Group name: `font-semibold text-gray-900 dark:text-gray-100`
- Member count badge: `text-xs text-gray-500 dark:text-gray-400` showing "(N accounts)"
- Expand/collapse chevron on the right

**Drag-and-drop interactions:**
- Drag an account into a group: account becomes a member, disappears from standalone list
- Drag an account out of a group: account becomes standalone again
- Drag a group to a different panel section: group's panel_type changes
- Drag an account between groups: moves membership
- All drag operations trigger immediate API calls (instant-save pattern from Phase 5)

## Interaction Contracts

### Group Row Expand/Collapse (Dashboard)
- **Trigger:** Click anywhere on the group row in PanelCard
- **Response:** Chevron rotates, member accounts slide in/out below the group row
- **Animation:** Chevron: `transition-transform duration-200`. Members: `max-height` transition, 200ms ease, `overflow: hidden`
- **State:** Multiple groups can be expanded simultaneously (Set-based state)
- **Motion preference:** All transitions respect `prefers-reduced-motion: reduce`
- **Keyboard:** Enter/Space toggles expansion

### Group Row Expand/Collapse (Settings)
- **Trigger:** Click on the chevron or group name area (not the drag handle)
- **Response:** Same animation pattern as Dashboard group row
- **State:** Multiple groups can be expanded simultaneously

### Donut Click Navigation
- **Trigger:** Click anywhere on the NetWorthDonut card
- **Response:** SPA navigation to `/net-worth` via `useNavigate()`
- **Visual feedback:** `cursor-pointer`, subtle shadow lift on hover
- **No page reload:** react-router-dom client-side navigation

### Time Range Selection
- **Trigger:** Click on a time range segment in the segmented control
- **Response:** Selected segment highlights (blue background, white text); chart and stats update
- **Data fetch:** New API call with `?days=N` parameter
- **Loading during fetch:** Chart shows previous data until new data arrives (no flash of loading state for fast responses); if fetch takes > 300ms, show a subtle opacity reduction on the chart area
- **Default:** 90d selected on page load

### Group Drag-and-Drop (Settings)
- **Trigger:** Drag handle on an account or group row
- **Response:** Item follows pointer; valid drop zones highlight
- **Drop zones:** Panel type sections (Liquid, Savings, Investments) and group rows
- **Drop feedback:** Instant API call on drop (instant-save pattern); optimistic UI update
- **Error:** Revert to previous position; show error toast
- **Mobile fallback:** Same touch drag pattern from Phase 5 AccountsSection

### Group Delete
- **Trigger:** Context action on a group row in Settings (could be a small trash icon or a right-click/long-press menu -- Claude's discretion)
- **Response:** Confirmation dialog appears
- **Confirm:** Group deleted, member accounts returned to standalone; API call fires
- **Cancel:** Dialog dismissed, no action
- **Animation:** Group row fades out on deletion
