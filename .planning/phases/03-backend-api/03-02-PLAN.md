  <name>Task 1: GET /api/balance-history handler with per-panel time series</name>
  <files>internal/api/handlers/history.go, internal/api/handlers/history_test.go</files>
  <behavior>
    - Test: returns three series (liquid, savings, investments) each as array of {date, balance} objects
    - Test: liquid for a given day = sum(checking balances that day) + sum(credit balances that day) where credit is negative
    - Test: days where only checking exists (no credit) still produce valid liquid entries (credit defaults to zero)
    - Test: savings series sums all savings account balances per day
    - Test: investments series sums all investment account balances per day
    - Test: "other" type accounts are excluded from all three series
    - Test: ?days=7 limits results to snapshots within the last 7 days
    - Test: invalid/negative days parameter is ignored (returns all data)
    - Test: empty history (no snapshots) returns {"liquid":[],"savings":[],"investments":[]}
    - Test: multiple accounts of the same type on the same day are summed correctly
  </behavior>
  <action>
    Create `internal/api/handlers/history.go`:

    1. Define unexported structs:
       - `balancePoint` with fields: Date string `json:"date"`, Balance string `json:"balance"`
       - `historyResponse` with fields: Liquid, Savings, Investments (all `[]balancePoint`, json tags: "liquid", "savings", "investments")

    2. Create `GetBalanceHistory(database *sql.DB) http.HandlerFunc` following factory pattern.

    3. Initialize response with empty slices: `[]balancePoint{}` for all three fields.

    4. Parse optional `days` query parameter:
       ```go
       days := 0
       if dStr := r.URL.Query().Get("days"); dStr != "" {
           if d, err := strconv.Atoi(dStr); err == nil && d > 0 {
               days = d
           }
       }
       ```

    5. Build SQL query to fetch all snapshots joined with account type:
       ```sql
       SELECT bs.balance_date, a.account_type, bs.balance
       FROM balance_snapshots bs
       JOIN accounts a ON a.id = bs.account_id
       WHERE a.account_type IN ('checking', 'credit', 'savings', 'investment')
       ```
       If days > 0, append: `AND bs.balance_date >= date('now', '-N days')` using fmt.Sprintf with the validated integer.
       Append: `ORDER BY bs.balance_date ASC, a.account_type`

    6. Iterate rows. For each row, parse balance with decimal.NewFromString(). Build a map[string]map of date -> {sumChecking, sumCredit, sumSavings, sumInvestments} using decimal accumulators.

    7. After processing all rows, iterate dates in sorted order. For each date:
       - liquid = sumChecking + sumCredit (credit already negative)
       - Append balancePoint{Date: date, Balance: liquid.StringFixed(2)} to Liquid slice
       - If sumSavings > 0 entries, append to Savings slice
       - If sumInvestments > 0 entries, append to Investments slice

       IMPORTANT: Only append to a panel's series if at least one account of that panel's types had a snapshot on that date. For liquid, append if either checking or credit had data. This avoids phantom "0.00" entries for panels with no data on a given day.

    8. Set Content-Type, encode response.

    Create `internal/api/handlers/history_test.go`:

    1. Reuse `setupFinanceTestDB` and seed helpers from summary_test.go (same test package).

    2. Write tests per behavior list. Use multiple days of snapshot data to verify time-series ordering and aggregation. Test the days parameter by seeding data across a range and verifying the filter.
  </action>

  </verify>

</task>
