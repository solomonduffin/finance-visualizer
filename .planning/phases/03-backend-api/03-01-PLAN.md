```sql
CREATE TABLE IF NOT EXISTS accounts (
    id           TEXT PRIMARY KEY,
    name         TEXT NOT NULL,
    account_type TEXT NOT NULL CHECK(account_type IN ('checking', 'savings', 'credit', 'investment', 'other')),
    currency     TEXT NOT NULL DEFAULT 'USD',
    org_name     TEXT,
    org_slug     TEXT,
    created_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS balance_snapshots (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    account_id   TEXT NOT NULL REFERENCES accounts(id),
    balance      TEXT NOT NULL,
    balance_date DATE NOT NULL,
    fetched_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(account_id, balance_date)
);

CREATE TABLE IF NOT EXISTS sync_log (
    id               INTEGER PRIMARY KEY AUTOINCREMENT,
    started_at       DATETIME NOT NULL,
    finished_at      DATETIME,
    accounts_fetched INTEGER DEFAULT 0,
    accounts_failed  INTEGER DEFAULT 0,
    error_text       TEXT
);
```

    - Test: liquid = sum(checking balances) + sum(credit balances) where credit values are already negative in DB (e.g., checking "1000.00" + credit "-200.00" = liquid "800.00")
    - Test: accounts with type "other" are excluded from all three panel totals
    - Test: when no accounts exist, all three fields return "0"
    - Test: last_synced_at is populated from most recent sync_log entry where error_text IS NULL and finished_at IS NOT NULL
    - Test: last_synced_at is null (JSON null) when no successful sync exists
    - Test: balance values are JSON strings not numbers (verify via raw JSON bytes check)
  </behavior>
  <action>
    Create `internal/api/handlers/summary.go`:

    1. Define unexported `summaryResponse` struct with fields: Liquid string, Savings string, Investments string, LastSyncedAt *string (json tags: "liquid", "savings", "investments", "last_synced_at").

    2. Create `GetSummary(database *sql.DB) http.HandlerFunc` following the factory pattern from settings.go.

    3. Query all accounts with their latest snapshot balance using a correlated subquery:
       ```sql
       SELECT a.account_type, bs.balance
       FROM accounts a
       JOIN balance_snapshots bs ON bs.account_id = a.id
       WHERE bs.balance_date = (
           SELECT MAX(bs2.balance_date)
           FROM balance_snapshots bs2
           WHERE bs2.account_id = a.id
       )
       ```

    4. Iterate rows, accumulate per-type sums using shopspring/decimal:
       - checking and credit balances accumulate into `liquid` (credit already negative, so Add works)
       - savings balances accumulate into `savings`
       - investment balances accumulate into `investments`
       - Skip "other" type entirely (per locked decision)

    5. Query last_synced_at:
       ```sql
       SELECT finished_at FROM sync_log
       WHERE error_text IS NULL AND finished_at IS NOT NULL
       ORDER BY id DESC LIMIT 1
       ```
       Use sql.NullString; set pointer if Valid, leave nil otherwise.

    6. Set Content-Type header, encode response with json.NewEncoder(w).Encode().

    7. Use `.StringFixed(2)` on decimal values for consistent "0.00" formatting (not "0").

    1. Create a `setupFinanceTestDB(t *testing.T) *sql.DB` helper (no password param needed, unlike auth tests). Uses same pattern: t.TempDir(), db.Open, db.Migrate, t.Cleanup.
