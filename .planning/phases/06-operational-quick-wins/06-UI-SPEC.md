**Phase-specific spacing rules:**

### GrowthBadge Specification

**Layout:** Inline span positioned immediately after the `formatCurrency(total)` text on the same line.

```
$12,450.00  [triangle_up] +2.3%
^Display    ^8px gap  ^text-sm font-medium
```

**States:**
1. **Positive growth** -- green triangle up + positive percentage with `+` sign
2. **Negative growth** -- red triangle down + negative percentage (sign inherent)
3. **No data / hidden** -- invisible placeholder (`visibility: hidden`) with identical dimensions
4. **Toggle OFF** -- invisible placeholder (same as no data)

**Tooltip:** Native `title` attribute. Format: `+$280.00 over 30 days` or `-$150.00 over 30 days`. Dollar sign, comma-separated thousands, 2 decimal places. Sign prefix always present.

**Invisible placeholder structure:** `<span class="invisible text-sm font-medium ml-2">[triangle_up] +0.0%</span>` -- reserves exact same horizontal and vertical space as a visible badge.

### SyncHistory Specification

**Layout:** Placed below the Accounts section on Settings page, inside a card with the same styling as other Settings cards (`bg-white dark:bg-gray-900 rounded-xl shadow-md p-6`).

**Section heading:** "Sync History" -- `text-lg font-medium` matching other card headings.

**Entry layout per sync:**
```
[status icon]  Mar 15, 2026 2:30 PM    12 accounts synced
[status icon]  Mar 14, 2026 2:30 PM    10 synced, 2 failed    [expand chevron]
```

**Status icons (inline SVG, 16x16):**
- Success: green circle with checkmark
- Partial: amber triangle with exclamation
- Failed: red circle with X

**Expandable error detail:**
- Triggered by clicking the entry row (for failed/partial entries only)
- Expand/collapse animation: CSS `max-height` transition with `overflow: hidden`, 200ms ease
- Error text rendered in `text-xs font-mono` with `bg-gray-100 dark:bg-gray-800 rounded p-3 mt-2`
- Sanitized server-side (defense-in-depth); no credentials or tokens visible

**Empty state:** "No sync history yet." -- `text-sm text-gray-500 dark:text-gray-400`

**Loading state:** "Loading sync history..." -- same styling as empty state

### DashboardPreferences Specification

**Layout:** Placed below the Sync History section on Settings page, inside a card with the same styling as other Settings cards.

**Section heading:** "Dashboard Preferences" -- `text-lg font-medium`.

**Toggle row:**
```
Show growth badges     [toggle switch]
```

**Toggle implementation:**
- Label: "Show growth badges" -- `text-sm font-medium text-gray-700 dark:text-gray-300`
- Switch: Custom CSS toggle (no library). 40px wide, 22px tall, rounded-full. Background `bg-gray-300 dark:bg-gray-600` when OFF, `bg-blue-600 dark:bg-blue-500` when ON. White circle knob (18px, 2px inset). Transition 150ms ease.
- Flex row: `flex items-center justify-between`

**Behavior:**
- ON by default for new users (server returns `true` when key absent)
- Instant save on toggle (Phase 5 instant-save pattern)
- Optimistic update: toggle switches immediately, reverts on API failure
- Error feedback: Toast notification on failure ("Failed to save preference")

## Interaction Contracts

### Growth Badge Hover
- **Trigger:** Mouse hover over the growth badge span
- **Response:** Native browser tooltip appears with dollar change and time period
- **Delay:** Browser default (typically ~500ms)
- **Touch:** No tooltip on touch devices (native `title` behavior)

### Sync Entry Expand/Collapse
- **Trigger:** Click anywhere on a sync entry row that has error details (partial or failed status)
- **Response:** Error detail panel slides open below the entry
- **Animation:** `max-height` from 0 to content height, `overflow: hidden`, 200ms ease transition
- **State:** Only one entry expanded at a time (clicking another collapses the current one)
- **Indicator:** Chevron icon rotates 90 degrees when expanded (matches AccountsSection hidden accounts pattern)
- **Cursor:** `cursor-pointer` on expandable rows

### Growth Badge Toggle
- **Trigger:** Click toggle switch in Dashboard Preferences
- **Response:** Toggle flips immediately (optimistic), API call fires
- **Success:** No feedback (silent success, consistent with Phase 5 pattern)
- **Failure:** Toggle reverts, error toast appears for 4 seconds
- **Effect on Dashboard:** Growth badges show/hide based on persisted state; Dashboard re-fetches or uses cached state on next load

### PanelCard Total Line Modification
**Before:**
```
<p class="text-2xl font-bold ...">$12,450.00</p>
```

**After:**
```
<p class="text-2xl font-bold ... flex items-baseline gap-2">
  <span>$12,450.00</span>
  <GrowthBadge ... />
</p>
```

The `flex items-baseline` ensures the smaller badge text aligns with the baseline of the larger total, not the center. The `gap-2` (8px) provides consistent spacing.

### Dashboard Data Fetching
Growth data added to the existing `Promise.all` batch in Dashboard.tsx:
```typescript
const [summaryData, accountsData, historyData, growthData] = await Promise.all([
  getSummary(),
  getAccounts(),
  getBalanceHistory(30),
  getGrowth(),  // NEW
])
```

The `getGrowth()` response includes `growth_badge_enabled` boolean to avoid an extra `getSettings()` call on the Dashboard.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Growth badge semantics | `title` attribute provides tooltip text for screen readers |
| Sync entry expand/collapse | `aria-expanded` attribute on clickable rows, `aria-controls` pointing to detail panel |
| Toggle switch | `role="switch"`, `aria-checked`, `aria-label="Show growth badges"` |
| Color alone not used | Growth direction conveyed by triangle direction (up/down) AND color (green/red) AND sign (+/-) |
| Keyboard navigation | Toggle responds to Space/Enter; sync entries respond to Enter/Space |
| Motion preference | Expand/collapse transition respects `prefers-reduced-motion: reduce` (no animation) |
