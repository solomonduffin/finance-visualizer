  <name>Task 2: Migration runner and full upfront schema</name>
  <files>internal/db/migrations.go, internal/db/migrations/000001_init.up.sql, internal/db/migrations/000001_init.down.sql, cmd/server/main.go</files>

    - TestMigrate_CreatesAccountsTable: Run Migrate, query sqlite_master for "accounts", assert exists with correct columns
    - TestMigrate_CreatesBalanceSnapshotsTable: Run Migrate, assert "balance_snapshots" exists with unique constraint on (account_id, balance_date)
    - TestMigrate_CreatesSyncLogTable: Run Migrate, assert "sync_log" table exists
    - TestMigrate_CreatesIndex: Run Migrate, assert idx_balance_snapshots_account_date index exists
    - TestMigrate_Idempotent: Run Migrate twice, assert no error on second run (ErrNoChange handled)
    - TestMigrate_AccountTypeCheck: Insert account with invalid type, assert CHECK constraint error
    - TestMigrate_BalanceSnapshotUniqueConstraint: Insert two snapshots with same account_id+balance_date, assert unique constraint violation
  </behavior>
  <action>
    1. Create `internal/db/migrations/000001_init.up.sql` with the full schema from RESEARCH.md:
       - `settings` table (key TEXT PRIMARY KEY, value TEXT NOT NULL)
       - `accounts` table with account_type CHECK constraint ('checking', 'savings', 'credit', 'investment', 'other'), currency default 'USD', org_name, org_slug, timestamps
       - `balance_snapshots` table with balance as TEXT (shopspring/decimal on read), UNIQUE(account_id, balance_date), foreign key to accounts
       - Index `idx_balance_snapshots_account_date` on (account_id, balance_date DESC)
       - `sync_log` table with started_at, finished_at, accounts_fetched, accounts_failed, error_text
       - NO explicit BEGIN/COMMIT (golang-migrate wraps implicitly)
    2. Create `internal/db/migrations/000001_init.down.sql`:
       - DROP TABLE IF EXISTS in reverse dependency order: balance_snapshots, sync_log, accounts, settings
       - DROP INDEX IF EXISTS idx_balance_snapshots_account_date
    3. Create `internal/db/migrations.go`:
       - `//go:embed migrations/*.sql` to embed migration files
       - `Migrate(dbPath string) error` using iofs.New + migrate.NewWithSourceInstance
       - URL scheme: `sqlite://` (NOT `sqlite3://`)
       - Import `_ "github.com/golang-migrate/migrate/v4/database/sqlite"` (NOT sqlite3)
       - Handle migrate.ErrNoChange gracefully (not an error)
    4. Create `cmd/server/main.go` as an entrypoint stub:
       - Loads config via config.Load()
       - Ensures data directory exists (os.MkdirAll)
       - Opens DB via db.Open(config.DBPath)
       - Runs migrations via db.Migrate(config.DBPath)
       - Logs success, placeholder for HTTP server (to be wired in Plan 02)
       - Uses log/slog for structured logging with tint colorizer for dev
    5. Write tests FIRST (RED), then implement (GREEN)
