<!-- Existing database schema (from 000001_init.up.sql) -->

```sql
CREATE TABLE IF NOT EXISTS settings (
    key   TEXT PRIMARY KEY,
    value TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS accounts (
    id           TEXT PRIMARY KEY,
    name         TEXT NOT NULL,
    account_type TEXT NOT NULL CHECK(account_type IN ('checking', 'savings', 'credit', 'investment', 'other')),
    currency     TEXT NOT NULL DEFAULT 'USD',
    org_name     TEXT,
    org_slug     TEXT,
    created_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS balance_snapshots (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    account_id   TEXT NOT NULL REFERENCES accounts(id),
    balance      TEXT NOT NULL,
    balance_date DATE NOT NULL,
    fetched_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(account_id, balance_date)
);

CREATE TABLE IF NOT EXISTS sync_log (
    id               INTEGER PRIMARY KEY AUTOINCREMENT,
    started_at       DATETIME NOT NULL,
    finished_at      DATETIME,
    accounts_fetched INTEGER DEFAULT 0,
    accounts_failed  INTEGER DEFAULT 0,
    error_text       TEXT
);
```

    - TestInferAccountType: "My Checking" -> checking, "Savings Account" -> savings, "Visa Credit Card" -> credit, "Brokerage" -> investment, "IRA" -> investment, "401k" -> investment, "Unknown" -> other
    - TestNextRunTime_Future: If target hour is later today, returns today at that hour
    - TestNextRunTime_Past: If target hour has passed today, returns tomorrow at that hour
    - TestSyncMutex: Concurrent calls to SyncOnce are serialized (second call does not start until first finishes)
  </behavior>
  <action>
    First, promote shopspring/decimal to a direct dependency:
    ```
    cd /home/solomon/finance-visualizer && go get github.com/shopspring/decimal
    ```

    Create `internal/sync/sync.go` with:

    1. `inferAccountType(name string) string` — keyword matching on lowercased name: "saving" -> savings, "credit"/"card" -> credit, "invest"/"brokerage"/"ira"/"401" -> investment, "check" -> checking, default -> other

    2. `nextRunTime(hour int) time.Time` — computes next occurrence of the given hour in local time; if that hour has already passed today, returns tomorrow at that hour

    3. Package-level `var syncMu sync.Mutex` to prevent concurrent SyncOnce executions

    4. `SyncOnce(ctx context.Context, db *sql.DB) error`:
       - Lock syncMu (defer Unlock)
       - Read `simplefin_access_url` from settings table; if empty, return nil (no-op, do NOT create sync_log entry)
       - Insert sync_log row with started_at=CURRENT_TIMESTAMP, get lastInsertId
       - Determine if this is first sync: `SELECT COUNT(*) FROM accounts`; if 0, set startDate to 30 days ago
       - Call `simplefin.FetchAccounts(accessURL, startDate)`
       - If FetchAccounts returns error, finalize sync_log with error_text, return error
       - For each account in result:
         - Parse balance with `decimal.NewFromString(account.Balance)`; if error, log + increment failed + continue
         - Derive balance_date: `time.Unix(account.BalanceDate, 0).UTC().Format("2006-01-02")`
         - Upsert account: `INSERT INTO accounts ... ON CONFLICT(id) DO UPDATE SET name=excluded.name, account_type=excluded.account_type, currency=excluded.currency, org_name=excluded.org_name, org_slug=excluded.org_slug, updated_at=CURRENT_TIMESTAMP`
         - Insert snapshot: `INSERT OR IGNORE INTO balance_snapshots (account_id, balance, balance_date) VALUES (?, ?, ?)`
         - Increment fetched on success
       - Finalize sync_log: UPDATE with finished_at, accounts_fetched, accounts_failed, error_text (nil if all ok)
       - Return nil

    5. `RunScheduler(ctx context.Context, syncHour int, db *sql.DB)`:
       - Infinite loop with `time.NewTimer(time.Until(nextRunTime(syncHour)))`
       - Select on ctx.Done() (stop timer, return) or timer.C (call SyncOnce, log errors via slog)
       - After each sync, loop back to recalculate next run time

    Tests use temp file DB (t.TempDir pattern from Phase 1) with db.Open + db.Migrate. Use httptest.Server for the SimpleFIN mock. Verify DB state after sync (query accounts, balance_snapshots, sync_log tables directly).

    Use module path: `github.com/solomon/finance-visualizer/internal/sync`
    Import SimpleFIN client: `github.com/solomon/finance-visualizer/internal/simplefin`
