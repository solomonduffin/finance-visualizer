    - TestSaveSettings_Success: POST with {access_url: "https://user:pass@host/simplefin"} saves to settings table, returns {ok: true}
    - TestSaveSettings_EmptyURL: POST with empty access_url returns 400
    - TestSaveSettings_TriggerSync: After saving a new access URL, SyncOnce is called (verify by checking sync_log for a new entry or use a flag)
    - TestSyncNow_Success: POST /api/sync/now returns {ok: true} immediately (non-blocking)
    - TestSyncNow_NoConfig: POST /api/sync/now when no access URL configured returns {ok: true} (SyncOnce no-ops gracefully)
  </behavior>
  <action>
    **Config extension** (`internal/config/config.go`):
    - Add `SyncHour int` field to Config struct
    - In Load(), read SYNC_HOUR env var; parse with strconv.Atoi; default to 6 if empty or invalid
    - Add test for SYNC_HOUR in config_test.go

    **Settings handlers** (`internal/api/handlers/settings.go`):

    1. `GetSettings(database *sql.DB) http.HandlerFunc`:
       - Query settings table for key=simplefin_access_url; set configured=true if non-empty
       - Query sync_log for most recent entry (ORDER BY id DESC LIMIT 1); extract finished_at and error_text
       - Return JSON: `{configured: bool, last_sync_at: string|null, last_sync_status: string|null}`
       - last_sync_status: null (no syncs), "success" (finished_at set, error_text null), or the error_text value

    2. `SaveSettings(database *sql.DB) http.HandlerFunc`:
       - Decode JSON body: `{access_url: string}`
       - Validate access_url is non-empty; return 400 if empty
       - Upsert into settings: `INSERT INTO settings (key, value) VALUES ('simplefin_access_url', ?) ON CONFLICT(key) DO UPDATE SET value=excluded.value`
       - Launch `go sync.SyncOnce(r.Context(), database)` to trigger immediate first sync
       - Return `{ok: true}` immediately (do not wait for sync to complete)
       - NOTE: Use `context.WithoutCancel(r.Context())` (Go 1.21+) or `context.Background()` for the goroutine so it is not cancelled when the HTTP response completes

    3. `SyncNow(database *sql.DB) http.HandlerFunc`:
       - Launch `go sync.SyncOnce(context.Background(), database)` in a goroutine
       - Return `{ok: true}` immediately
       - The sync mutex in the sync package prevents concurrent runs; if a sync is already running, the goroutine will block on the mutex until it finishes — this is acceptable behavior

    All handlers follow the existing factory pattern: `func HandlerName(database *sql.DB) http.HandlerFunc`

    Tests use temp file DB (t.TempDir pattern) with db.Open + db.Migrate. For SaveSettings sync trigger test, verify by checking sync_log after a brief sleep or by querying the settings table to confirm the URL was saved.
