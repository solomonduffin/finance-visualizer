# Phase 8 -- UI Design Contract

> Visual and interaction contract for the alert system: expression builder, alert management page, status badges, enable/disable toggles, expandable history, and SMTP configuration in Settings. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-rolled Tailwind components) |
| Icon library | inline SVG (16x16, stroke-based, matching SyncHistory/DashboardPreferences pattern) |
| Font | System font stack (Tailwind default: ui-sans-serif, system-ui, sans-serif) |

**Note:** Continues the hand-rolled Tailwind convention established in v1.0 and maintained through Phases 6-7. Phase 8 adds 2 new pages/sections (Alerts page, Email Configuration in Settings) and 2 new components (AlertRuleForm, AlertRuleCard). No shadcn or third-party component library.

**Focal points:**
- Alerts page: Rule cards with at-a-glance status badges are the primary visual element. The inline expression builder form is the primary interaction element.
- Settings Email Configuration: Standard form card matching existing SimpleFIN Connection card pattern.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, `gap-1` |
| sm | 8px | Compact element spacing, operand row gaps, `gap-2`, `py-2` |
| md | 16px | Default section padding, card internal spacing, `p-4`, `mb-4` |
| lg | 24px | Card padding, section card `p-6`, `mb-6` |
| xl | 32px | Not heavily used in this phase |
| 2xl | 48px | Page vertical padding, `py-12` |
| 3xl | 64px | Not used in this phase |

Exceptions: none

**Phase-specific spacing rules:**
- Operand row vertical gap: `gap-2` (8px) between stacked operand rows in the expression builder
- Builder section gap: `gap-4` (16px) between the "When", "Comparison", and "Actions" sections of the builder form
- Alert rule card padding: `p-5` (20px) -- matches PanelCard convention for card-level padding
- Alert history indent: `pl-4` (16px) left padding for expanded history entries within a rule card
- Rule card list gap: `gap-4` (16px) between alert rule cards in the list
- Settings SMTP form field gap: `mb-4` (16px) between form field groups, matching SimpleFIN Connection card pattern
- Inline builder expansion: `mt-4` (16px) gap between the "+ New Alert" button and the expanded builder form

## AlertRuleForm Specification

**Purpose:** Inline expression builder form for creating and editing alert rules.

**Layout structure:**
```
[Rule Name input]

When:
  [+ dropdown: Liquid Balance] [+]
  [- dropdown: Credit Card    ] [-] [x]
  [Add operand button]

is:
  [operator dropdown: <] [threshold input: 5000]

Options:
  [x] Notify on recovery

[Save Rule] [Cancel]
```

**Rule name input:**
- Full-width text input at top of form
- Label: "Rule Name" (`text-sm font-semibold`)
- Placeholder: "e.g., Low cash warning"
- Style: matches existing Settings input pattern -- `w-full px-3 py-2 border border-gray-300 dark:border-gray-600 rounded-lg text-gray-900 dark:text-gray-100 bg-white dark:bg-gray-800 placeholder-gray-400 dark:placeholder-gray-500 focus:outline-none focus:ring-2 focus:ring-blue-500 focus:border-transparent`

**"When" section (operand rows):**
- Section label: "When" (`text-sm font-semibold text-gray-700 dark:text-gray-300 mb-2`)
- Each operand row is a horizontal flex container: `flex items-center gap-2`
- First operand row has no operator prefix (implicitly "+")
- Subsequent operand rows start with a +/- toggle button: small pill button showing "+" or "-", click to toggle
  - Style: `w-8 h-8 flex items-center justify-center rounded-md text-sm font-semibold border border-gray-300 dark:border-gray-600 text-gray-700 dark:text-gray-300 hover:bg-gray-100 dark:hover:bg-gray-700 transition-colors`
- Operand dropdown: `<select>` element with grouped optgroups
  - Optgroup "Buckets": Liquid Balance, Savings Balance, Investments Balance, Net Worth
  - Optgroup "Groups": (dynamically populated from GET /api/groups)
  - Optgroup "Accounts": (dynamically populated from GET /api/accounts, using display names)
  - Style: `flex-1 px-3 py-2 border border-gray-300 dark:border-gray-600 rounded-lg bg-white dark:bg-gray-800 text-sm text-gray-900 dark:text-gray-100 focus:outline-none focus:ring-2 focus:ring-blue-500`
- Remove button (all rows except the first): small "x" icon button
  - Style: `w-8 h-8 flex items-center justify-center rounded-md text-gray-400 hover:text-red-500 hover:bg-red-50 dark:hover:bg-red-900/20 transition-colors`
  - SVG: 16x16 X icon (stroke-based)
- Add operand button: text-only link style below the operand rows
  - Text: "+ Add term"
  - Style: `text-sm font-semibold text-blue-600 dark:text-blue-400 hover:text-blue-700 dark:hover:text-blue-300 cursor-pointer transition-colors`

**"is" section (comparison):**
- Section label: "is" (`text-sm font-semibold text-gray-700 dark:text-gray-300 mb-2`)
- Horizontal flex: `flex items-center gap-2`
- Operator dropdown: `<select>` with options: <, <=, >, >=, ==
  - Style: same as operand dropdown but with `w-20` fixed width
- Threshold input: `<input type="text" inputMode="decimal">`
  - Placeholder: "0.00"
  - Style: `flex-1 px-3 py-2 border border-gray-300 dark:border-gray-600 rounded-lg bg-white dark:bg-gray-800 text-sm text-gray-900 dark:text-gray-100 focus:outline-none focus:ring-2 focus:ring-blue-500`

**Options section:**
- "Notify on recovery" checkbox using custom toggle switch (matching DashboardPreferences pattern)
  - Row layout: `flex items-center justify-between`
  - Label: "Notify on recovery" (`text-sm font-semibold text-gray-700 dark:text-gray-300`)
  - Toggle: reuse exact toggle switch pattern from DashboardPreferences (`role="switch"`, `aria-checked`, blue-600 on, gray-300 off)

**Action buttons:**
- Horizontal flex: `flex items-center gap-3 mt-4`
- "Save Rule" button (primary CTA):
  - Style: `bg-blue-600 text-white py-2 px-4 rounded-lg font-semibold text-sm hover:bg-blue-700 focus:outline-none focus:ring-2 focus:ring-blue-500 focus:ring-offset-2 disabled:opacity-50 disabled:cursor-not-allowed transition-colors`
  - Loading state: "Saving..." with `disabled`
- "Cancel" button:
  - Style: `py-2 px-4 rounded-lg text-sm font-semibold text-gray-600 dark:text-gray-400 hover:bg-gray-100 dark:hover:bg-gray-700 transition-colors`

**Form container:**
- Renders inline below the "+ New Alert" button or in place of a rule card when editing
- Container: `bg-white dark:bg-gray-800 rounded-xl shadow-md p-5 border border-gray-200 dark:border-gray-700`
- Sections separated by `space-y-4` (16px)

**Validation:**
- Name required: red border on input + error text "Rule name is required" below input
- At least 1 operand required: "Add at least one term" error below the operand section
- Threshold required: red border + "Threshold is required"
- Validation error text: `text-xs text-red-600 dark:text-red-400 mt-1`

## AlertRuleCard Specification

**Layout:** Card row for each alert rule, rendered in a vertical list on the Alerts page.

**Card container:** `bg-white dark:bg-gray-800 rounded-xl shadow-md p-5 border border-transparent dark:border-gray-700` (matches PanelCard)

**Collapsed state (default):**
```
[v] Low cash warning                      [Normal]  [toggle] [...]
    Liquid Balance - Credit Card < $5,000
    Last checked: 2 minutes ago
```

**Row 1 -- header row:** `flex items-center justify-between`
- Left side: `flex items-center gap-2`
  - Chevron icon: 16x16 stroke SVG, right-pointing when collapsed, rotated 90 degrees when expanded. Reuse exact SVG from SyncHistory.tsx
  - Rule name: `text-sm font-semibold text-gray-900 dark:text-gray-100 truncate`
- Right side: `flex items-center gap-3`
  - Status badge: pill badge
    - Normal: `bg-green-100 text-green-800 dark:bg-green-900 dark:text-green-200 px-2 py-0.5 rounded-full text-xs font-semibold`
    - Triggered: `bg-red-100 text-red-800 dark:bg-red-900 dark:text-red-200 px-2 py-0.5 rounded-full text-xs font-semibold`
  - Toggle switch: identical to DashboardPreferences toggle (`role="switch"`, `aria-checked`, `w-10 h-[22px]`, blue-600 on, gray-300 off)
  - Actions menu: icon button with 3-dot vertical ellipsis SVG (16x16)
    - Style: `p-1.5 rounded-lg text-gray-400 hover:text-gray-600 dark:hover:text-gray-300 hover:bg-gray-100 dark:hover:bg-gray-700 transition-colors`
    - Dropdown menu on click: `absolute right-0 mt-1 bg-white dark:bg-gray-800 border border-gray-200 dark:border-gray-700 rounded-lg shadow-lg py-1 z-10`
      - "Edit" option: `px-4 py-2 text-sm text-gray-700 dark:text-gray-300 hover:bg-gray-100 dark:hover:bg-gray-700 cursor-pointer`
      - "Delete" option: `px-4 py-2 text-sm text-red-600 dark:text-red-400 hover:bg-red-50 dark:hover:bg-red-900/20 cursor-pointer`

**Row 2 -- expression summary:** `text-sm text-gray-500 dark:text-gray-400 mt-1`
- Format: human-readable summary of the operand expression
- Example: "Liquid Balance - Credit Card < $5,000"
- Rendering: join operand labels with their operators, then append comparison operator and formatted threshold
- Truncate with `truncate` class if expression is very long, with `title` attribute showing full text

**Row 3 -- metadata:** `text-xs text-gray-500 dark:text-gray-400 mt-1`
- Format: "Last checked: {timeAgo}" or "Never checked" if `last_eval_at` is null
- Show last computed value: "Value: $4,850" if `last_value` is present

**Expanded state (history):**
- Toggled by clicking the chevron or rule name area
- Animation: chevron `transition-transform duration-200 motion-reduce:transition-none`, history list `max-height` transition 200ms ease `overflow-hidden motion-reduce:transition-none` (matching SyncHistory and GroupRow pattern)
- History section: `pl-4 mt-3 border-t border-gray-100 dark:border-gray-700 pt-3`
- Each history entry: `flex items-center gap-2 py-1`
  - State label: "Triggered" or "Recovered"
    - Triggered: `text-xs font-semibold text-red-600 dark:text-red-400`
    - Recovered: `text-xs font-semibold text-green-600 dark:text-green-400`
  - Value: `text-xs text-gray-600 dark:text-gray-300` showing "at $4,850"
  - Timestamp: `text-xs text-gray-500 dark:text-gray-400` showing relative time
- Empty history: `text-xs text-gray-500 dark:text-gray-400 italic` showing "No events yet"
- Max entries displayed: 10 most recent (sorted by notified_at DESC)

**States:**
1. **Enabled + Normal:** Green badge, toggle on, full opacity
2. **Enabled + Triggered:** Red badge, toggle on, full opacity
3. **Disabled:** Gray badge text "Disabled" (`bg-gray-100 text-gray-600 dark:bg-gray-800 dark:text-gray-400`), toggle off, card at reduced opacity: `opacity-60`
4. **Editing:** Card replaced by AlertRuleForm pre-filled with rule values

## Interaction Contracts

### "+ New Alert" Button
- **Trigger:** Click the "+ New Alert" button at the top of the Alerts page
- **Response:** AlertRuleForm slides into view below the header, button hides
- **Animation:** Form fades in with a subtle slide-down (`opacity 0->1, translateY -8px->0`, 200ms ease-out). Motion-reduce: instant display.
- **Cancel:** Clicking "Cancel" collapses the form, restores the "+ New Alert" button

### Expression Builder Operand Management
- **Add operand:** Click "+ Add term" appends a new operand row with "+" operator and first available dropdown option
- **Remove operand:** Click "x" button on a row removes it (minimum 1 row always present; remove button hidden on last remaining row)
- **Toggle operator:** Click the +/- button on a row toggles between "+" and "-"
- **Dropdown selection:** Standard `<select>` behavior; options grouped by optgroups

### Alert Rule Toggle (Enable/Disable)
- **Trigger:** Click the toggle switch on a rule card
- **Response:** Immediate optimistic UI update (toggle flips, card opacity changes for disabled). API call fires in background (PATCH /api/alerts/:id with enabled: true/false)
- **Revert on failure:** Toggle flips back, error toast shown
- **Accessibility:** `role="switch"`, `aria-checked`, `aria-label` dynamically set

### Alert Rule Card Expand/Collapse
- **Trigger:** Click the chevron icon or the rule name
- **Response:** Chevron rotates, history section slides in/out below the card content
- **Animation:** Chevron `transition-transform duration-200`, history `max-height` transition 200ms ease, `overflow: hidden`
- **State:** Multiple cards can be expanded simultaneously (Set-based state, matching GroupRow pattern)
- **Motion preference:** All transitions respect `prefers-reduced-motion: reduce`
- **Keyboard:** Enter/Space on the chevron/name toggles expansion

### Alert Rule Edit
- **Trigger:** Click "Edit" from the actions dropdown menu
- **Response:** Card content is replaced by AlertRuleForm pre-filled with the rule's current values
- **Save:** Form submits PUT /api/alerts/:id, on success card returns to view mode with updated data
- **Cancel:** Form collapses, card returns to previous view mode

### Alert Rule Delete
- **Trigger:** Click "Delete" from the actions dropdown menu
- **Response:** Inline confirmation appears below the card or replaces the dropdown
- **Confirm:** API DELETE /api/alerts/:id fires, card fades out and is removed from list
- **Cancel:** Confirmation dismissed, no action
- **Animation:** Card fade-out 200ms on deletion. Motion-reduce: instant removal.

### SMTP Test Email
- **Trigger:** Click "Test Email" button in Settings Email Configuration section
- **Response:** Button shows "Sending..." in disabled state. API POST /api/email/test fires with current form values (even unsaved)
- **Success:** Inline success message appears below button, auto-clears after 5 seconds
- **Failure:** Inline error message with server error detail appears below button, persists until next action

### SMTP Save
- **Trigger:** Click "Save" button in Settings Email Configuration section
- **Response:** Button shows "Saving..." in disabled state. API POST /api/email/config fires
- **Success:** Inline success message appears below button, auto-clears after 5 seconds
- **Failure:** Inline error message appears below button

### Actions Dropdown Menu
- **Trigger:** Click the 3-dot icon on a rule card
- **Response:** Small dropdown menu appears anchored to the button
- **Close:** Click outside the menu, or press Escape, or click a menu item
- **Keyboard:** Tab cycles through menu items, Escape closes

## Expression Summary Rendering

**How to display complex multi-term expressions as readable text in rule cards:**

- Join operand labels with their arithmetic operators
- Append comparison operator and formatted threshold
- Use display names for accounts, group names for groups, bucket labels for buckets

**Examples:**
- Single bucket: "Liquid Balance < $5,000"
- Bucket minus account: "Liquid Balance - Visa Credit Card < $5,000"
- Multiple terms: "Liquid Balance + Savings Balance - Emergency Fund >= $10,000"
- Net worth: "Net Worth < $50,000"
- Account group: "Coinbase > $1,000"

**Truncation:** If summary exceeds card width, apply `truncate` CSS class with full text in `title` attribute.
